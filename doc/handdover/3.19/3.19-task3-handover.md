## 本轮完成的变更

### 重大设计变更：废弃套餐概念

- `therapy_packages` 和 `therapy_package_items` 表已 DROP
- `prescriptions.therapy_package_id` 字段已删除
- 处方直接关联 `therapy_project`，通过 `prescription_steps` 表实现
- AI 推荐和医生开处方都是直接给具体的多个 project

### Migration 文件

1. `supabase/migrations/20260319101933_restructure_therapy_multi_room.sql` — 新增 `prescription_steps`、`system_config`、修改现有表
2. `supabase/migrations/20260319110104_drop_therapy_packages_and_cleanup.sql` — DROP 套餐表、去掉 prescriptions 的 package_id

### Seed Data 脚本（v3）

- `scripts/gen-therapy-data.mjs`：
  - 过滤空 name 行（不再把通用禁忌症当 project）
  - 不再生成 packages/package_items 数据
  - 新增禁忌症聚合逻辑：项目级 + 全局通用 + 睡眠区通用 → 合并到每个 project 的 contraindications JSONB
  - 生成 contraindications 字典表 seed（13 条）
- `supabase/seed.sql`：去掉套餐引用和旧禁忌症数据
- `little_tool/supabase/db-reset.sh`：验证查询已更新

### 禁忌症设计

- 全局通用禁忌症（所有 project 都有）：听力损失、耳鸣、急性精神病状态、严重心律失常
- 睡眠区通用禁忌症（仅睡眠区 project）：身体疼痛/疾病、创伤患者
- 每个 project 的 `contraindications` JSONB = 项目自身禁忌症 + 适用的通用禁忌症
- `contraindications` 字典表存所有去重词条，供前端搜索录入
