**分诊-生理测量终端**（护士/分诊台）

设备：联网刷卡器 + 电脑
权限：创建新用户、写入生理读数

| 步骤 | 功能 | 对应 Block 思路 |
|------|------|----------------|
| 1. 测量血压与心率 | 接入设备读数或手动录入生理数据（血压、心率） | `VitalSignsInput` — 生理数据录入表单 |
| 2. 刷医保卡，新用户进入系统 | 刷卡读取医保信息，自动创建/匹配患者档案 | `PatientCheckIn` — 刷卡签到 + 患者信息创建/匹配 |
| 3. 用户进入队列 | 患者进入候诊队列，分配到对应科室/医生 | `QueueAssignment` — 队列分配确认 |

---

**医生-医生终端**（门诊医生）

设备：电脑
权限：读取排队表、读写生理参数、读写禁忌症、读写量表信息

| 步骤 | 功能 | 对应 Block 思路 |
|------|------|----------------|
| 1. 叫号系统叫到用户 | 从候诊队列中叫号，加载患者信息 | `CallQueue` — 叫号面板，显示候诊列表 + 叫号按钮 |
| 2. 刷医保卡，读取用户信息、生理信息 | 刷卡后自动加载患者档案 + 分诊台录入的生理数据 | `PatientInfoBar`（已有）— 扩展显示生理数据 |
| 3. 禁忌症询问与录入 | 内置禁忌症字典，通过首字母检索快速录入 | `ContraindicationInput` — 禁忌症快速检索 + 录入（字典联想） |
| 4. 医生通过询问收集量表信息 | 结构化量表问卷填写 | `ScaleForm` — 量表数据采集表单 |
| 5. 数据上传至云端AI，生成处方建议 | 调用AI接口，返回处方建议（不泄露屏幕信息给用户） | `AISuggestionPanel` — AI处方建议面板（仅医生可见） |
| 6. 医生调整处方参数 | 基于AI建议，医生手动调整处方 | `PrescriptionForm`（已有）— 扩展支持AI建议预填 |
| 7. 医生下处方 | 最终确认并提交处方（就是我们已有的处方页面） | `HerbGrid` + `ActionBar`（已有） |
| 8. 更新用户状态，进入治疗队列 | 患者从诊疗队列转入治疗队列，上传处方方案 | `StatusTransition` — 状态流转确认 + 处方上传 |

---

**治疗-护士终端**（治疗室护士）

设备：电脑
权限：读取治疗排队表、读取用户/生理/处方/禁忌症信息、读写治疗后生理和量表信息

| 步骤 | 功能 | 对应 Block 思路 |
|------|------|----------------|
| 1. 治疗系统叫到用户 | 从治疗队列中叫号 | `TreatmentQueue` — 治疗叫号面板 |
| 2. 刷医保卡，读取全部信息 | 加载患者信息 + 生理信息 + 处方信息 + 禁忌症 | `TreatmentPatientView` — 治疗患者综合信息视图 |
| 3. 开始治疗 | 确认开始治疗，记录治疗开始时间 | `TreatmentAction` — 治疗执行操作面板 |
| 4. 治疗完毕，测试心跳、血压并记录 | 治疗后生理数据录入 | `PostVitalSigns` — 治疗后生理数据录入 |
| 5. 护士询问患者问题，填写量表 | 治疗后量表评估 | `PostScaleForm` — 治疗后量表采集 |
| 6. 更新队列状态，用户出队 | 患者完成治疗，从队列移除 | `QueueComplete` — 出队确认 |
