## 步骤 1：需求与数据分析

明确变更的业务意图，定位涉及的数据表和字段。

- 用一句话描述变更目标
- 对照 `doc/backend/数据模块设计.md`，列出涉及的现有表
- 确认是否需要新增表
- 确认数据来源（Excel/CSV/手动录入/API）
- 确认是否有跨表关联变更（外键、级联）

输出格式：
📋 需求与数据分析

变更目标：xxx
涉及现有表：xxx
新增表：xxx（如有）
数据来源：xxx
跨表关联：xxx
