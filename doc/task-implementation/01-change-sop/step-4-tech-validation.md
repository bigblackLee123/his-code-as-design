## 步骤 4：技术先验

对于引入新技术、新依赖或复杂流程（如外部 AI 模型、硬件交互）的变更，必须先进行独立的技术验证。

- **目的**: 隔离风险，确保核心技术路径跑通，避免在主项目中引入不确定性。
- **操作**:
  - 创建一个独立的验证脚本或命令行工具（如 `.ts` ,`.js` 文件或python脚本）。
  - 在脚本中完成最小闭环的核心流程。
  - **示例**: 如果要引入 AI 模型阅读诊疗建议，此脚本需要能：
    1.  获取模拟的患者信息。
    2.  调用模型 API 并上传信息。
    3.  接收并打印模型的输出。
- **产出**:
  - 一个可独立运行并验证成功的脚本。
  - 确认技术方案可行性的结论。

输出格式：
```
🔬 技术先验
- 是否需要：是/否
- 验证方案：[简述脚本验证的流程]
- 验证结论：成功/失败
```
