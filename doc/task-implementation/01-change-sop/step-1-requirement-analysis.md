## 步骤 1：需求分析

明确变更的业务意图和技术范围。

- 用一句话描述变更目标（做什么、为什么）
- 确认变更涉及的页面（哪个 `pages/` 目录）
- 确认变更涉及的 Block（哪些 `blocks/*.tsx` 文件）
- 如果涉及新页面，确认适用的页面模板类型（列表页/详情页/表单页/仪表盘页，参考 `his-patterns.md`）

输出格式：
```
📋 需求分析
- 变更目标：xxx
- 涉及页面：xxx
- 涉及 Block：xxx（新增/修改/删除）
- 页面模板：xxx（如适用）
```
