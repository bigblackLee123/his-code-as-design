# 3.20 测试计划：数据链路逐步验证

> 基于数据流图 12 步，对每一步的 DB 状态做精确断言。
> 执行方式：通过 Supabase MCP `execute_sql` 逐步查询验证。
> 前置条件：supabase local 运行中，DB 已 reset + seed。

---

## 测试策略

每一步操作后，立即查询相关表，验证：
1. 记录是否正确创建/更新
2. 字段值是否符合预期
3. 状态机流转是否正确
4. 关联关系（FK）是否正确建立
5. 不该变的数据是否保持不变（回归）

---

### TC-01：分诊签到 — patients 创建

**操作**：分诊终端输入医保卡号签到（或手动建档）

**验证 SQL**：
```sql
SELECT id, name, gender, age, insurance_card_no, status, created_at
FROM patients
WHERE insurance_card_no = '<测试卡号>';
```

**断言**：
- [ ] 记录存在且唯一
- [ ] status = `checked-in`
- [ ] name / gender / age / insurance_card_no 字段非空
- [ ] created_at 为当前时间附近

### TC-02：生理数据采集 — vital_signs (pre-treatment)

**操作**：分诊护士录入血压、心率

**验证 SQL**：
```sql
SELECT vs.id, vs.consultation_id, vs.stage,
       vs.systolic_bp, vs.diastolic_bp, vs.heart_rate,
       vs.recorded_at, vs.recorded_by
FROM vital_signs vs
JOIN consultations c ON c.id = vs.consultation_id
WHERE c.patient_id = '<patient_id>'
  AND vs.stage = 'pre-treatment';
```

**断言**：
- [ ] 记录存在
- [ ] stage = `pre-treatment`
- [ ] systolic_bp 在 40-300 范围内
- [ ] diastolic_bp 在 20-200 范围内
- [ ] heart_rate 在 20-300 范围内
- [ ] consultation_id 正确关联到该患者的 consultation
- [ ] recorded_by 非空

### TC-03：入候诊队列 — queue_items (waiting) + patient status

**操作**：分诊完成，患者入候诊队列

**验证 SQL**：
```sql
-- 队列记录
SELECT qi.id, qi.patient_id, qi.consultation_id,
       qi.queue_type, qi.queue_number, qi.status, qi.enqueued_at
FROM queue_items qi
WHERE qi.patient_id = '<patient_id>'
  AND qi.queue_type = 'waiting';

-- 患者状态
SELECT status FROM patients WHERE id = '<patient_id>';

-- consultation 状态
SELECT status FROM consultations WHERE patient_id = '<patient_id>';
```

**断言**：
- [ ] queue_items 记录存在，queue_type = `waiting`，status = `waiting`
- [ ] queue_number > 0 且递增
- [ ] consultation_id 非空
- [ ] patients.status = `waiting`
- [ ] consultations.status = `created`

### TC-05：禁忌症录入 — consultation_contraindications

**操作**：医生选择禁忌症（即时持久化）

**验证 SQL**：
```sql
SELECT cc.consultation_id, cc.contraindication_id,
       c.name AS contraindication_name, c.code
FROM consultation_contraindications cc
JOIN contraindications c ON c.id = cc.contraindication_id
WHERE cc.consultation_id = '<consultation_id>';
```

**断言**：
- [ ] 记录数 = 医生选择的禁忌症数量
- [ ] contraindication_id 对应正确的禁忌症字典记录
- [ ] 重复选择/取消后，记录数与最终选择一致（DELETE + INSERT 策略）

### TC-07B：处方保存 — prescriptions + prescription_steps

**操作**：医生确认处方并流转

**验证 SQL**：
```sql
-- 处方主记录
SELECT p.id, p.consultation_id, p.meta, p.created_at
FROM prescriptions p
WHERE p.consultation_id = '<consultation_id>';

-- 处方步骤
SELECT ps.id, ps.prescription_id, ps.project_id,
       tp.name AS project_name, ps.region,
       ps.sort_order, ps.status,
       ps.started_at, ps.completed_at, ps.treatment_record_id
FROM prescription_steps ps
JOIN therapy_projects tp ON tp.id = ps.project_id
WHERE ps.prescription_id = '<prescription_id>'
ORDER BY ps.sort_order;
```

**断言**：
- [ ] prescriptions 记录存在且唯一
- [ ] prescription_steps 数量 = 医生选择的疗愈项目数
- [ ] 每个 step 的 region 与对应 therapy_project.region 一致
- [ ] 所有 step 的 status = `pending`
- [ ] started_at / completed_at / treatment_record_id 均为 NULL
- [ ] sort_order 从 1 开始递增

### TC-08：房间签到（以睡眠区为例）

**操作**：睡眠区终端输入卡号后四位签到

**验证 SQL**：
```sql
-- 患者状态应变为 treating
SELECT status FROM patients WHERE id = '<patient_id>';

-- 本房间的 treatment_record 已创建
SELECT id, consultation_id, region, start_time, end_time, duration
FROM treatment_records
WHERE consultation_id = '<consultation_id>'
  AND region = '睡眠区';

-- 本房间的 steps 状态变为 in-progress
SELECT ps.id, tp.name, ps.region, ps.status, ps.started_at
FROM prescription_steps ps
JOIN therapy_projects tp ON tp.id = ps.project_id
WHERE ps.prescription_id = '<prescription_id>'
  AND ps.region = '睡眠区';
```

**断言**：
- [ ] patients.status = `treating`
- [ ] treatment_records 存在一条，region = `睡眠区`，start_time 非空，end_time 为 NULL
- [ ] 本房间所有 steps 的 status = `in-progress`，started_at 非空
- [ ] 其他房间的 steps 仍为 `pending`

### TC-09：本房间治疗完成

**操作**：护士点击「本房间完成」

**验证 SQL**：
```sql
-- treatment_record 更新
SELECT id, start_time, end_time, duration
FROM treatment_records
WHERE consultation_id = '<consultation_id>'
  AND region = '睡眠区';

-- steps 状态更新
SELECT ps.id, tp.name, ps.status, ps.completed_at, ps.treatment_record_id
FROM prescription_steps ps
JOIN therapy_projects tp ON tp.id = ps.project_id
WHERE ps.prescription_id = '<prescription_id>'
  AND ps.region = '睡眠区';

-- 全局 steps 状态汇总
SELECT ps.region, ps.status, COUNT(*) AS cnt
FROM prescription_steps ps
WHERE ps.prescription_id = '<prescription_id>'
GROUP BY ps.region, ps.status
ORDER BY ps.region;
```

**断言**：
- [ ] treatment_records: end_time 非空，duration > 0（秒）
- [ ] 本房间所有 steps: status = `completed`，completed_at 非空
- [ ] 本房间所有 steps: treatment_record_id = 该房间的 treatment_record.id
- [ ] 其他房间 steps 仍为 `pending`

### TC-12：完成出队 — 最终状态全量校验

**操作**：确认出队

**验证 SQL**：
```sql
-- 患者最终状态
SELECT status FROM patients WHERE id = '<patient_id>';

-- consultation 最终状态
SELECT status FROM consultations WHERE patient_id = '<patient_id>';

-- 治疗队列最终状态
SELECT status FROM queue_items
WHERE patient_id = '<patient_id>' AND queue_type = 'treatment';

-- 全部 steps 最终状态
SELECT ps.status, COUNT(*) AS cnt
FROM prescription_steps ps
WHERE ps.prescription_id = '<prescription_id>'
GROUP BY ps.status;

-- treatment_records 数量 = 涉及的房间数
SELECT COUNT(*) AS room_count
FROM treatment_records
WHERE consultation_id = '<consultation_id>';
```

**断言**：
- [ ] patients.status = `completed`
- [ ] consultations.status = `completed`
- [ ] queue_items (treatment).status = `completed`
- [ ] 所有 prescription_steps.status = `completed`
- [ ] treatment_records 数量 = 处方涉及的不同 region 数

## 全量数据完整性校验（最终汇总）

一次就诊结束后，跑这个汇总查询确认所有关联数据齐全：

```sql
SELECT
  p.name AS patient_name,
  p.status AS patient_status,
  c.status AS consultation_status,
  (SELECT COUNT(*) FROM vital_signs WHERE consultation_id = c.id) AS vital_signs_count,
  (SELECT COUNT(*) FROM scale_results WHERE consultation_id = c.id) AS scale_results_count,
  (SELECT COUNT(*) FROM consultation_contraindications WHERE consultation_id = c.id) AS contraindication_count,
  (SELECT COUNT(*) FROM ai_suggestions WHERE consultation_id = c.id) AS ai_suggestion_count,
  (SELECT COUNT(*) FROM prescriptions WHERE consultation_id = c.id) AS prescription_count,
  (SELECT COUNT(*) FROM prescription_steps ps
   JOIN prescriptions rx ON rx.id = ps.prescription_id
   WHERE rx.consultation_id = c.id) AS step_count,
  (SELECT COUNT(*) FROM prescription_steps ps
   JOIN prescriptions rx ON rx.id = ps.prescription_id
   WHERE rx.consultation_id = c.id AND ps.status = 'completed') AS completed_step_count,
  (SELECT COUNT(*) FROM treatment_records WHERE consultation_id = c.id) AS treatment_record_count,
  (SELECT status FROM queue_items WHERE patient_id = p.id AND queue_type = 'treatment' LIMIT 1) AS treatment_queue_status
FROM patients p
JOIN consultations c ON c.patient_id = p.id
WHERE p.id = '<patient_id>';
```

**预期结果**：

| 字段 | 预期值 |
|------|--------|
| patient_status | completed |
| consultation_status | completed |
| vital_signs_count | 2（pre + post） |
| scale_results_count | 2（pre + post） |
| contraindication_count | ≥ 0（取决于患者） |
| ai_suggestion_count | 1 |
| prescription_count | 1 |
| step_count | = 选择的项目数 |
| completed_step_count | = step_count |
| treatment_record_count | = 涉及的房间数 |
| treatment_queue_status | completed |
```
