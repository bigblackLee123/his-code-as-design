## 任务 1：过号功能

**现状：** CallQueue 只有「叫号」按钮，叫号后直接进入就诊。无法跳过某个患者。

**目标：** 在叫号队列中，医生可以对当前排队的患者执行「过号」操作，将其移到队列末尾。

**改动文件：**
- `src/services/supabase/waitingQueueService.ts` — 新增 `skipPatient(queueItemId)` 方法
- `src/services/mock/queueService.ts` — 新增 mock 实现
- `src/pages/doctor-terminal/blocks/CallQueue.tsx` — 每个队列项增加「过号」按钮

**实现要点：**
- `skipPatient`：将指定 queue_item 的 queue_number 更新为当前最大值 + 1，效果等同于重新排队
- CallQueue 每行增加一个小按钮（SkipForward 图标），点击后调用 skipPatient 并刷新队列
- 过号后 Realtime 自动刷新

**目标：** 每个区域（睡眠区/情志区/运动疗愈区）最多只能选 1 个项目。选择同区域新项目时自动替换旧的。

**改动文件：**
- `src/pages/doctor-terminal/blocks/TherapyProjectSelector.tsx` — handleAdd 增加同区域替换逻辑

**实现要点：**
- `handleAdd` 中检查 selectedProjects 是否已有同 region 的项目
- 如果有，替换（移除旧的 + 添加新的）
- 下拉列表中已选区域的其他项目显示为「替换」而非「已选」

## 任务 7：AI 建议流程提示 + 流式回答

**现状：** AISuggestionPanel 点击「获取建议」后只有一个 loading spinner，然后一次性显示完整结果。

**目标：**
1. 流程提示：loading 过程中分阶段显示提示（数据整合完毕 → 提交云端处理 → 云端知识搜集 → 生成方案中…）
2. 流式回答：AI 的 reason 文本流式逐字显示，而非一次性展示

**改动文件：**
- `src/pages/doctor-terminal/blocks/AISuggestionPanel.tsx` — 流程提示 UI + 流式文本渲染
- `src/services/supabase/aiService.ts` — 改为 SSE/流式调用
- `supabase/functions/ai-therapy-suggestion/index.ts` — 改为流式响应
- `src/services/mock/aiService.ts` — mock 流式效果

**实现要点：**
- 前端阶段提示：用 setTimeout 模拟阶段切换（因为实际阶段在后端，前端用时间估算）
  - 0s: 「数据整合完毕」
  - 1s: 「提交云端处理」
  - 3s: 「云端知识搜集中…」
  - 收到首个 token: 「方案生成中…」
- Edge Function 改为 SSE 流式返回（直连模式用 stream: true）
- 前端用 EventSource 或 fetch + ReadableStream 消费流
- reason 文本逐字渲染，projectIds/confidence 等结构化数据在流结束后解析
