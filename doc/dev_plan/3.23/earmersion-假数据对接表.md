# Earmersion 新增界面 — 数据对接表

> 静区/动区面板均使用现有 therapy_projects 字段，无需扩展表结构。
> 仅传感器数据和报告/知识库部分涉及假数据或后续对接。

---

## 1. StaticZonePanel（静区干预面板）

当 region = `"睡眠区"` 或 `"情志区"` 时展示。全部使用已有字段，零新增。

| 面板位置 | 展示内容 | 数据来源 | 字段 |
|---------|---------|---------|------|
| 栏1 标题 | "节律参数" | — | — |
| 栏1 主值 | BPM 数值 | ✅ therapy_projects | `bpm` |
| 栏1 副文 | 能量等级 | ✅ therapy_projects | `energyLevel`（如"深度镇静"） |
| 栏2 标题 | "情绪导向" | — | — |
| 栏2 主值 | 情绪标签 | ✅ therapy_projects | `mood`（如"平静"） |
| 栏2 副文 | 作用机制摘要 | ✅ therapy_projects | `mechanism`（截取前 20 字） |
| 栏3 标题 | "执行时长" | — | — |
| 栏3 主值 | 计时器 MM:SS | ✅ treatmentState | `startTime` 计算 |
| 脉冲动画圆 | 治疗中播放动画 | ✅ treatmentState | `status === "treating"` |
| 当前配方条 | 配方名 + BPM | ✅ patient.projects | `name`, `bpm` |

---

## 2. ActiveZonePanel（动区康复面板）

当 region = `"运动疗愈区"` 时展示。BPM 用真数据，传感器为假数据。

| 面板位置 | 展示内容 | 数据来源 | 字段 |
|---------|---------|---------|------|
| BPM 滑块主值 | 目标步态节拍 | ✅ therapy_projects | `bpm` |
| BPM 滑块副文 | 作用机制 | ✅ therapy_projects | `mechanism` |
| 传感器-步幅对称性 | -- | ✅ 占位 | 显示 `--`，设备未接入 |
| 传感器-节奏同步率 | -- | ✅ 占位 | 显示 `--`，设备未接入 |
| 传感器-平衡稳定度 | -- | ✅ 占位 | 显示 `--`，设备未接入 |
| 训练阶段进度 | 热身→基础→强化→放松 | ✅ 前端推算 | 从 bpm ±20 推算，纯展示 |
| 当前配方条 | 配方名 | ✅ patient.projects | `name` |

---

## 3. TreatmentReport（结诊报告面板）

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 报告编号 | ✅ 前端生成 | `EM-{YYYYMMDD}-{4位随机}` 格式，如 `EM-20260324-A3F7` |
| 生成时间 | ✅ 前端 | `new Date()` |
| 治疗时长 | ✅ treatmentState | startTime/endTime |
| 配方信息 | ✅ patient.projects | name/bpm/mood/mechanism |
| 生理数据对比 | ✅ 已有 | preVitals vs postVitals |
| 量表结果 | ✅ 已有 | scaleResults |
| 家庭康复建议 | ❌ 假数据 | 硬编码模板文案，后续接 AI |

---

## 4. AIKnowledgeCard（AI 知识库匹配卡）

| 字段 | 数据来源 | 说明 |
|------|---------|------|
| 文献列表 | ✅ 硬编码 | 固定 4 条文献，后续接百炼知识库动态检索 |
| 模型状态 | ✅ 硬编码 | 固定"就绪"，后续接 aiService 状态 |

---

## 汇总

| 类别 | 状态 | 说明 |
|------|------|------|
| 静区面板 | ✅ 全部真数据 | 零新增字段 |
| 动区面板-BPM/配方 | ✅ 真数据 | |
| 动区面板-传感器 | ✅ 占位 `--` | 设备未接入，如实显示 |
| 动区面板-训练阶段 | ✅ 前端推算 | |
| 结诊报告 | ✅ 基本全真 | 编号前端生成，家庭建议硬编码模板 |
| AI 知识库卡 | ✅ 硬编码 | 后续接百炼 |

> 唯一的"假数据"只剩：家庭康复建议的模板文案。其余全部真数据或合理占位。
