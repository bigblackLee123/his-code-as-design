## 待办任务

### 任务一：症状拼音首字母搜索输入（前端）

医生终端需要一个症状输入组件，支持拼音首字母模糊匹配。

- 交互模式：Combobox / Tag Input，输入拼音首字母实时过滤候选症状，选中后以 Tag 形式展示
- 症状词库来源：`曙光医院-音乐疗愈空间内容梳理-2.xlsx` 套餐表「匹配症状」列汇总去重
- 技术方案：pinyin-pro 库提取首字母 → 前端本地过滤（数据量小，无需后端搜索）
- 参考实践：类似中文输入法联想，医疗 HIS 常见的拼音首字母检索模式

### 任务四：百炼平台 AI 推荐对接（前后端联调）

打通「患者数据 → AI 分析 → 推荐方案 → 前端展示」链路。

- 对接阿里云百炼平台 API
- 输入：患者当前生理数据（血压/心率/体温等）+ 量表评分 + 历史诊疗记录
- 输出：推荐疗愈套餐 + 推荐理由
- 前端：医生终端展示 AI 推荐卡片，医生可采纳/修改/忽略
- 需要讨论：prompt 设计、知识库构建、模型选择
