## Part D — 治疗终端功能补全

目标：Hook 拆分 → 视觉改造 → Page 重组，对齐 demo Stage 3-5。

### Phase 1：Hook 拆分

将混在视觉组件里的逻辑抽成独立 hook，为后续新建面板做准备。

| 步骤 | 新建文件 | 从哪里抽出 | 职责 | 状态 |
|------|---------|-----------|------|------|
| D0.1 | `useTreatmentTimer.ts` | TreatmentAction.tsx | 计时器：elapsed / running / start / stop / reset | ✅ 完成 |
| D0.2 | `useTreatmentFlow.ts` | TreatmentTerminalPage.tsx | 状态机流转：pageStatus + 所有 handler | ✅ 完成 |
| D0.3 | `useRoomComplete.ts` | RoomCompleteCheck.tsx | 房间完成 API 调用 + 状态管理 | ✅ 完成 |
| D0.4 | `usePostVitals.ts` | PostVitalSigns.tsx | 表单值 + 验证 + 保存逻辑 | ✅ 完成 |

### Phase 2：视觉改造

| 步骤 | 文件 | 说明 | 状态 |
