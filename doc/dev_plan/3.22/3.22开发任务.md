# 3.22 开发任务：前端架构重构 — Block 自治化

## 背景

当前三个终端页面的 service 调用存在两种混用模式：
- 模式 A（Block 自治）：Block 内部调 service，通过回调传结果给 Page
- 模式 B（Page 代劳）：Page 在 handler 里调 service，Block 只是纯 UI

混用导致「业务逻辑到底在哪」不清晰，维护成本随功能增长会快速上升。

## 目标架构

```
Page（纯编排层）
  ├── BlockA.tsx（UI + hook 消费）
  │     └── useBlockA.ts（service 调用 + loading/error + 状态管理）
  ├── BlockB.tsx
  │     └── useBlockB.ts
  └── ...
```

**Page 只做三件事：**
1. 管状态机（当前处于哪个步骤 / pageStatus）
2. 编排 Block 的渲染顺序和条件
3. 在 Block 之间传递必要的 ID（patientId、consultationId）

**Block 自治原则：**
- 自己的 service 自己调
- 自己的 loading / error 自己管
- 通过 props 回调只向上传「业务结果」（不传中间态）
- 业务逻辑超过 30 行就拆到 `useXxx.ts` hook，Block 文件只留 JSX
- 每个 Block 文件 ≤ 200 行

### 需要重构的 4 处

| # | 问题 | 现状 | 目标 |
|---|------|------|------|
| 1 | ContraindicationInput 持久化在 Page | DoctorTerminalPage.handleContraindicationChange 调 contraindicationService.saveForConsultation + consultationHelper | 移入 Block 内部，props 传入 patientId，Block 自己获取 consultationId 并持久化 |
| 2 | ScaleForm 持久化在 Page | DoctorTerminalPage.handleScaleSubmit 调 scaleService.saveResult | 移入 useScaleForm.ts（已存在），props 传入 patientId，hook 内部调 saveResult |
| 3 | AI 建议采纳后的项目解析在 Page | DoctorTerminalPage.handleAdoptSuggestion 调 therapyService.getProjectById 逐个查项目 | 移入 AISuggestionPanel 内部，onAdopt 回调直接传 TherapyProject[]（而非只传 suggestion） |
| 4 | 治疗终端 handleCheckIn 调 consultationHelper | TreatmentTerminalPage.handleCheckIn 调 consultationHelper.getActiveId | 移入 TreatmentQueue 内部，onCheckIn 回调额外带上 consultationId |

## 实施步骤

### 步骤 1：ContraindicationInput 自治化

**修改文件：**
- `src/pages/doctor-terminal/blocks/ContraindicationInput.tsx`
- `src/pages/doctor-terminal/DoctorTerminalPage.tsx`

**新建文件：**
- `src/pages/doctor-terminal/blocks/useContraindicationInput.ts`

**改动要点：**
- ContraindicationInput props 新增 `patientId: string`
- 新建 useContraindicationInput hook：
  - 内部管理 `items: Contraindication[]` state
  - 搜索：调 `contraindicationService.search()`
  - 持久化：选择/移除时调 `consultationHelper.getActiveId()` → `contraindicationService.saveForConsultation()`
  - 暴露 `items`, `handleAdd`, `handleRemove`, `search` 等
- DoctorTerminalPage 删除 `handleContraindicationChange` 中的 service 调用，只保留 state 同步（供其他 Block 读取）

### 步骤 2：ScaleForm 持久化移入 useScaleForm

**修改文件：**
- `src/pages/doctor-terminal/blocks/useScaleForm.ts`
- `src/pages/doctor-terminal/blocks/ScaleForm.tsx`
- `src/pages/doctor-terminal/DoctorTerminalPage.tsx`

**改动要点：**
- ScaleForm props 新增 `patientId: string`
- useScaleForm 新增持久化逻辑：提交时调 `scaleService.saveResult(patientId, result, 'pre')`
- DoctorTerminalPage.handleScaleSubmit 删除 service 调用，只保留 `setConsultationData`

### 步骤 3：AISuggestionPanel 返回完整项目

**修改文件：**
- `src/pages/doctor-terminal/blocks/AISuggestionPanel.tsx`
- `src/pages/doctor-terminal/DoctorTerminalPage.tsx`

**改动要点：**
- AISuggestionPanel 内部：采纳时自己调 `therapyService.getProjectById()` 解析项目
- onAdopt 回调签名从 `(suggestion: AITherapySuggestion) => void` 改为 `(suggestion: AITherapySuggestion, projects: TherapyProject[]) => void`
- DoctorTerminalPage.handleAdoptSuggestion 删除 therapyService 调用，直接用回调传入的 projects

### 步骤 4：TreatmentQueue 返回 consultationId

**修改文件：**
- `src/pages/treatment-terminal/blocks/TreatmentQueue.tsx`
- `src/pages/treatment-terminal/TreatmentTerminalPage.tsx`

**改动要点：**
- TreatmentQueue 内部：checkInByRoom 成功后，自己调 `consultationHelper.getActiveId()` 获取 consultationId
- onCheckIn 回调签名从 `(checkIn: RoomCheckIn) => void` 改为 `(checkIn: RoomCheckIn, consultationId: string) => void`
- TreatmentTerminalPage.handleCheckIn 删除 consultationHelper 调用，直接用回调传入的 consultationId

### 步骤 5：清理 DoctorTerminalPage imports

**修改文件：**
- `src/pages/doctor-terminal/DoctorTerminalPage.tsx`

**改动要点：**
- 删除不再需要的 import：`contraindicationService`, `scaleService`, `therapyService`, `consultationHelper`
- 只保留 `patientService`（Mock 预填逻辑需要，后续 Mock 逻辑也可以抽走）
- Page 文件应该只 import Block 组件 + types

### 步骤 6：清理 TreatmentTerminalPage imports

**修改文件：**
- `src/pages/treatment-terminal/TreatmentTerminalPage.tsx`

**改动要点：**
- 删除 `consultationHelper` import
- Page 文件只 import Block 组件 + types
