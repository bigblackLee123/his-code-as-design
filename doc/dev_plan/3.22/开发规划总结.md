## 🔄 数据流总结

```
分诊终端                    医生终端                     治疗终端
┌──────────┐  入候诊队列   ┌──────────┐  入治疗队列    ┌──────────┐
│ 患者签到  │ ──────────→ │ 叫号诊疗  │ ──────────→  │ 叫号治疗  │
│ 生理数据  │  Realtime   │ 量表/禁忌  │  Realtime    │ 计时/出队  │
└──────────┘  队列同步    │ 套餐处方  │  队列同步    └──────────┘
                          └──────────┘
```

**关键流程：**
1. 患者在分诊签到 → 创建 Consultation + 入候诊队列
2. 医生叫号 → 患者状态变为 consulting
3. 医生填写禁忌症、症状、量表 → 获取 AI 建议 → 选择疗愈项目
4. 医生确认处方 → 创建 Prescription + 拆解为 PrescriptionSteps → 患者入治疗队列
5. 治疗护士按房间刷卡签到 → 开始本房间治疗
6. 完成本房间 → 检查是否还有其他房间
7. 全部房间完成 → 采集治疗后数据 → 出队 → Consultation 完成

## 📝 技术要点

### 禁忌症清单模式
- 使用 `contraindicationService.getAll()` 获取全部禁忌症
- 按 `category` 分组显示
- 每项为 Checkbox，勾选时调用 `handleAdd()`
- 搜索框用于过滤列表

### AI 流程提示
- 创建步骤指示器组件：`StepIndicator.tsx`
- 根据 `consultationData` 判断各步骤完成状态
- 未完成步骤显示为灰色 + 禁用

### 流式回答
- Edge Function 返回 `Transfer-Encoding: chunked`
- 前端使用 `response.body.getReader()` 逐块读取
- 每块包含 JSON 对象：`{ reason: "...", done: false }`
- 最后一块：`{ reason: "...", done: true, projectIds: [...], projectNames: [...] }`

### 治疗提示区域
- 创建 `TreatmentHintBox.tsx` 组件
- 根据 `pageStatus` 显示对应提示
- 使用 Alert 或 Info 组件样式

### 房间冲突检查
- 在 `checkInByRoom()` 中查询：
  ```sql
  SELECT * FROM treatment_records 
  WHERE consultation_id = ? AND region = ? AND end_time IS NULL
  ```
- 如有结果则抛出错误
