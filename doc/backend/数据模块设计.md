# HIS 数据模块设计

> 曙光医院信息系统（HIS）后端数据架构。基于 Supabase (PostgreSQL)。
> 最后更新：2026-03-20

---

## 模块一：核心基建与患者 (Base & Patient)

### patients（患者主表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| name | VARCHAR(50) | 姓名 |
| gender | VARCHAR(10) | 性别 |
| age | INTEGER | 年龄 |
| phone | VARCHAR(20) | 手机号 |
| id_number | VARCHAR(30) | 身份证号 |
| insurance_card_no | VARCHAR(50), UNIQUE | 医保卡号 |
| status | patient_status ENUM | 流程状态，见状态机章节 |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间（自动触发器） |

### contraindications（禁忌症字典表 - 预置数据）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| code | VARCHAR(20), UNIQUE | 禁忌症编码 |
| name | VARCHAR(100) | 禁忌症名称 |
| pinyin | VARCHAR(200) | 拼音全拼（搜索用） |
| pinyin_initial | VARCHAR(50) | 拼音首字母（快捷搜索用） |
| category | VARCHAR(50) | 分类 |

### scale_templates（量表模板）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| name | VARCHAR(100) | 量表名称 |
| description | TEXT | 量表描述 |

### scale_questions（量表题目）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| template_id | UUID, FK → scale_templates | 所属模板 |
| sort_order | INTEGER | 排序序号 |
| text | TEXT | 题目内容 |
| type | question_type ENUM | single-choice / multi-choice / slider / text |
| required | BOOLEAN | 是否必填 |
| options | JSONB | 选项列表 [{value, label, score}] |
| slider_config | JSONB | 滑块配置 {min, max, step}，仅 slider 类型 |

## 模块三：业务流转核心 (The Hub)

### consultations（就诊主记录 - 串联分诊、医生、治疗）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| patient_id | UUID, FK → patients | 患者 |
| status | VARCHAR(30) | 状态：created → in-progress → completed |
| created_at | TIMESTAMPTZ | 创建时间 |

### consultation_contraindications（就诊-禁忌症关联表）

| 字段 | 类型 | 说明 |
|------|------|------|
| consultation_id | UUID, FK → consultations | 就诊 ID |
| contraindication_id | UUID, FK → contraindications | 禁忌症 ID |
| PK | (consultation_id, contraindication_id) | 联合主键 |

写入时机：医生选择禁忌症时即时写入（AI 聚合数据依赖此表）。

### vital_signs（体征数据）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| consultation_id | UUID, FK → consultations | 就诊 ID |
| stage | vital_stage ENUM | pre-treatment（分诊前）/ post-treatment（治疗后） |
| systolic_bp | INTEGER | 收缩压 40-300 mmHg |
| diastolic_bp | INTEGER | 舒张压 20-200 mmHg |
| heart_rate | INTEGER | 心率 20-300 次/分 |
| recorded_at | TIMESTAMPTZ | 记录时间 |
| recorded_by | VARCHAR(50) | 记录人 |

### scale_results（量表结果）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| consultation_id | UUID, FK → consultations | 就诊 ID |
| stage | scale_stage ENUM | pre（诊前）/ post（治疗后） |
| template_id | UUID, FK → scale_templates | 量表模板 |
| answers | JSONB | 答卷 {questionId: value} |
| total_score | INTEGER | 总分 |
| submitted_at | TIMESTAMPTZ | 提交时间 |

写入时机：医生提交量表时即时写入（AI 聚合数据依赖此表）。

### ai_suggestions（AI 疗愈建议记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| consultation_id | UUID, FK → consultations, UNIQUE | 就诊 ID（一次就诊一条建议） |
| model | VARCHAR(50) | 调用的模型名或百炼 App ID |
| prompt_snapshot | TEXT | 发送给 AI 的完整 prompt（审计用） |
| raw_response | TEXT | AI 原始返回文本（审计 + 改进分析用） |
| herbs | JSONB | 历史遗留字段，当前未使用。推荐数据存于 raw_response 中解析 |
| usage | TEXT | 历史遗留字段 |
| notes | TEXT | AI 推荐理由 |
| confidence | DECIMAL(3,2) | 置信度 0-1 |
| generated_at | TIMESTAMPTZ | 生成时间 |

持久化方式：Edge Function 调用百炼后自动 upsert，保存 prompt 快照和原始返回，供后续分析改进。

### prescriptions（处方）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| consultation_id | UUID, FK → consultations, UNIQUE | 就诊 ID |
| meta | JSONB | 处方元数据（兼容保留，疗愈处方可为空） |
| herbs | JSONB | 兼容保留字段，疗愈处方的项目明细通过 prescription_steps 关联 |
| total_amount | DECIMAL(10,2) | 总金额 |
| created_at | TIMESTAMPTZ | 创建时间 |

注意：疗愈处方的核心数据在 `prescription_steps` 表，prescriptions 作为处方主记录存在，meta/herbs 为兼容保留。

## 模块四：队列与治疗执行

### queue_items（候诊/治疗队列）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| consultation_id | UUID, FK → consultations | 就诊 ID |
| patient_id | UUID, FK → patients | 患者 ID |
| queue_type | queue_type ENUM | waiting（候诊）/ treatment（治疗） |
| queue_number | INTEGER | 排队序号 |
| status | queue_status ENUM | waiting / in-progress / completed |
| region | VARCHAR(50) | 治疗队列时标记目标房间（候诊队列为空） |
| enqueued_at | TIMESTAMPTZ | 入队时间 |

Realtime：已启用（REPLICA IDENTITY FULL + supabase_realtime publication）。

### treatment_records（治疗执行记录 - 每个房间一条）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| consultation_id | UUID, FK → consultations | 就诊 ID（1:N，非 UNIQUE） |
| region | VARCHAR(50) | 本次治疗的房间 |
| start_time | TIMESTAMPTZ | 开始时间 |
| end_time | TIMESTAMPTZ | 结束时间 |
| duration | INTEGER | 时长（秒） |
| created_at | TIMESTAMPTZ | 创建时间 |

### prescription_steps（处方执行步骤 - 多房间流转核心）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID, PK | 主键 |
| prescription_id | UUID, FK → prescriptions | 处方 ID |
| project_id | UUID, FK → therapy_projects | 疗愈项目 ID |
| region | VARCHAR(50) | 睡眠区/情志区/运动疗愈区（冗余，方便终端按房间过滤） |
| sort_order | INTEGER | 项目在处方中的顺序 |
| status | prescription_step_status ENUM | pending / in-progress / completed / skipped |
| started_at | TIMESTAMPTZ, NULLABLE | 开始时间 |
| completed_at | TIMESTAMPTZ, NULLABLE | 完成时间 |
| treatment_record_id | UUID, FK → treatment_records, NULLABLE | 关联的治疗记录 |

Realtime：待启用（3.20 任务，多房间终端需实时感知 steps 状态变化）。

## 数据实体关系

patients │ ├──1:N──▶ consultations (就诊记录，串联一次完整流程) │ ├──1:1──▶ ai_suggestions │ ├──1:1──▶ prescriptions │ │ └──1:N──▶ prescription_steps (多房间流转核心) │ ├──M:N──▶ contraindications (通过 consultation_contraindications) │ ├──1:N──▶ vital_signs (分诊前 + 治疗后) │ ├──1:N──▶ scale_results (诊前量表 + 治疗后量表) │ └──1:N──▶ treatment_records (每个房间一条) │ └──1:N──▶ queue_items (候诊队列 + 治疗队列)

独立字典/内容表: contraindications (禁忌症字典，预置数据) therapy_projects (疗愈项目库，含 contraindications JSONB) scale_templates → scale_questions (量表模板) system_config (系统配置，API Key 等)

## 状态机

### Patient.status

checked-in ──▶ waiting ──▶ consulting ──▶ pending-treatment ──▶ treating ──▶ completed 分诊签到 入候诊队列 医生叫号 处方确认/入治疗队列 房间签到 全部治疗完成


### Consultation.status

created ──▶ in-progress ──▶ completed 创建 医生接诊 治疗完成出队


### QueueItem.status

候诊队列: waiting ──▶ in-progress ──▶ (出队，患者转入治疗队列) 治疗队列: waiting ──▶ in-progress ──▶ completed


### PrescriptionStep.status

pending ──▶ in-progress ──▶ completed ──▶ skipped（跳过）
